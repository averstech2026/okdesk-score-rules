# Типовые проблемы → база баллов (черновик на утверждение)

Маппинг для формулы (без выезда):

```
баллы = (база × N + осложнение*) × (1|2 дежурство)
* осложнение только если complication_level и непустое описание
выезд → только 60 (typical не суммируется)
```

Поля Okdesk: `object_count` (N), `complication_level`, `complication`.  
Тип **Нестандарт** + `nonstandard_detail` — для дыр каталога (не строка этой таблицы).

Статус: **черновик** — правим веса с тимлидами, потом вносим имена в Okdesk.

## 2. Статьи базы знаний — как оценивать

Уже есть поле `knowledge_base_issue` (ссылка/id статьи). Отдельный **тип заявки** для БЗ **не нужен** — достаточно пункта в typical (тип «Нестандарт» оставляем для дыр каталога проблем).

### Рекомендация

| | Решение |
|---|---------|
| Что завести в typical | **«Статья базы знаний»** |
| База | **C15 = 15** |
| N | всегда **1** (не пакет объектов) |
| Обязательно | заполнить `knowledge_base_issue` (ссылка на статью) + в описании/комментарии что сделали (новая / правка) |
| Тяжёлая статья | база 15 + осложнение **+15** или **+30** (с описанием: «новая развёрнутая статья по X») |
| Выезд | если вдруг с выездом — по общему правилу **только 60**, typical не суммируется |

### Почему не отдельный тип заявки

- Баллы уже идут через заявку + typical.  
- Тип «Нестандарт» занят дырами каталога.  
- Поле БЗ уже есть — привязка статьи без нового типа.  
- В отчётах легко фильтр: `typical = Статья базы знаний`.

### Альтернатива (если захотите жёстче)

Два пункта typical вместо одного:

| Typical | База |
|---------|-----:|
| Статья БЗ: правка / дополнение | 5 (T5) |
| Статья БЗ: новая развёрнутая | 30 (S30) |

Пока предлагаем **один пункт C15** + осложнение при необходимости — меньше пунктов в списке.

### Антифрод

- Нет ссылки в `knowledge_base_issue` → не засчитывать C15 (или красный в отчёте).  
- Одна и та же статья на десяток заявок → аудит.  
- «Статья» без текста в БЗ → снять баллы.

## 3. Что не смешивать

| Typical | Не путать с |
|---------|-------------|
| ЧЗ оперативный | Расследование / расхождение |
| Оплата не проходит | Банк / модуль банка (настройка ПО) |
| НДС точечный | Массовое обновление НДС |
| Раскатка ПО | Автообновление: сбой / доводка |
| Статья БЗ | Консультация (просто ответили в заявке) |

## 4. Лист утверждения весов

| # | Вопрос | Решение |
