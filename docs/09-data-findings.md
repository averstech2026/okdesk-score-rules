# Заметки по данным Okdesk (6 месяцев)

Источник: [avers.okdesk.ru](https://avers.okdesk.ru/), выгрузка REST API `completed`/`closed` с 23.01.2026 по 22.07.2026.

Сырые данные: `data/` (не в git). Отчёт: `analysis/report.md`.

## Поля, важные для правил

| Код | Название | Тип | Замечание |
|-----|----------|-----|-----------|
| `ticket_weight` | Баллы | select | Свободный выбор из длинного списка (5…1000), не категория работ |
| `typical` | Типовая проблема | select (обяз.) | Лучшая основа для справочника категорий |
| `solution_method` | Способ решения | multiselect (обяз.) | Для эталонов и аудита |
| `departure` | Выезд | checkbox | Сейчас редко заполнен относительно ожиданий |
| `watch` | Дежурство | checkbox | Ставится вручную (риск рассинхрона с табелем) |
| `complication_level` | Осложнение | **Значение из списка** | `+15` / `+30`; надбавка только с заполненным `complication` |
| `complication` | Описание осложнения | **Строка** | пустое → надбавка 0 + красный в отчёте |
| `object_count` | Число объектов (N) | **Строка** (число) | пакет: база × N |
| `nonstandard_detail` | Суть нестандарта | **Строка** | только тип «Нестандарт» |
| `knowledge_base_issue` | Статья в БЗ | string | Связь с базой знаний |
| `pos_type` | Тип кассы | select | КСО / Фронт / Офис |

## Ключевые факты (черновик анализа)

- ~**19 640** решённых/закрытых заявок за ~6 мес (~3 000/мес).
- Баллы проставлены у **~75%** (без баллов ~4 900).
- Доминируют **5** и **10** баллов; «классические» 15/30 встречаются реже; есть хвост до **500–1000**.
- Явные **конвейеры**: высокая доля одного клиента у отдельных исполнителей (IFCM, MFC и др.).
- Поле `typical` + `solution_method` уже дают каркас для «тип работ → норма баллов» вместо свободного `ticket_weight`.

## Как обновить выгрузку

```bash
python3 scripts/fetch_issues.py 6          # список + карточки (кэш в data/issues)
python3 scripts/analyze_scores.py          # analysis/report.md + summary.json
```

Токен только в локальном `.env` (см. `.env.example`).
