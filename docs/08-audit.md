# Аудит и флаги аномалий

При ~1000 заявок/мес ручная проверка всего невозможна. Контроль — **правила + выборка + разбор выбросов**.

## Обязательная / приоритетная выборка

- Все заявки с надбавками и/или суммой (без ×2 и без выезда) ≥ порога (ориентир 45).
- Случайные 5–10% остальных за месяц.
- Все спорные категории по жалобе/возврату.

## Флаги для аналитики (основа будущего инструмента)

| Флаг | Смысл |
|------|--------|
| Доля дорогих категорий выше медианы сегмента | Возможен завышенный микс |
| Доля надбавок выше медианы | Риск «осложнение на всё» |
| Короткое время при высокой категории | База/надбавка не бьётся с фактом |
| Пик баллов в конце месяца | Риск добора объёма |
| ×2 вне табеля дежурства | Ошибка или обход |
| Много выездов `V_CLIENT` / `V_OTHER` | Выезды без техпричины |
| Высокие баллы при слабом качестве (возвраты) | Объём без результата |
| Смешение: человек в пуле по роли, а баллы как с конвейера (или наоборот) без объяснения | Ошибка сегментации / учёта |

«Фейковую» заявку на 100% автоматикой не доказать — флаги дают **очередь на разбор**, не автоматический приговор.
