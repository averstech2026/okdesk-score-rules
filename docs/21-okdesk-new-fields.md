## Чеклист создания в Okdesk

### 1. Осложнение
1. Настройки → Дополнительные атрибуты → Новый  
2. Код: `complication_level`  
3. Название: `Осложнение`  
4. Тип: **Значение из списка**  
5. Значения: `+15`, `+30`  
6. Привязать к обычным типам заявок (не к MFC-bulk, если отдельный)

### 2. Описание осложнения
1. Код: `complication`  
2. Название: `Описание осложнения`  
3. Тип: **Строка**  
4. Те же типы заявок, что у п.1  

### 3. Число объектов (N)
1. Код: `object_count`  
2. Название: `Число объектов (N)`  
3. Тип: **Строка**  
4. Подсказка: для раскаток/пакетов; сверить со списком в описании  

### 4. Нестандарт — это **тип заявки**, не статус / не «этап»

«Только на этом типе» = в форме атрибута поле **«Типы заявок»** → выбрать только «Нестандарт».  
Это **не** про «Заполнять при входе в статус».

| Что | Где в Okdesk | Название в UI | Код |
|-----|--------------|---------------|-----|
| **Тип заявки** | Настройки → **Типы заявок** → Новый | **Нестандарт** | `nonstandard` |
| **Атрибут** | Доп. атрибуты → Новый | **Суть нестандарта** | `nonstandard_detail` |

**Зачем:** вместо пункта «Другое» в typical. Инженер выбирает тип заявки «Нестандарт» и пишет одной фразой, что за проблема. Потом по этим формулировкам дополняем каталог.

**Как завести атрибут** (сначала создайте тип «Нестандарт»):

1. Код: `nonstandard_detail`  
2. Название: **`Суть нестандарта`**  
3. Тип атрибута: **Строка**  
4. **Типы заявок:** только «Нестандарт» (не все 15, как у Осложнения / N)  
5. Клиентский портал — не показывать  
6. MFC этим типом не оформлять  

Поля 1–3 (Осложнение, Описание, N) — на обычных типах. На «Нестандарт» главное поле — **Суть**.

## Формула баллов

```
если departure (Выезд):
  баллы = 60 × (1|2 watch)
иначе:
  баллы = (база(typical) × N(object_count или 1)
           + осложнение если complication_level И complication непустое
          ) × (1|2 watch)
```

| Поле | Влияет на баллы? |
|------|------------------|
| Осложнение + Описание | да, только вместе |
| Число объектов (N) | да, на пакетах |
| Суть нестандарта | нет (учёт / каталог) |
| Выезд | да → только 60 |
