## Ответ: «что такое авто?» (f.3)

**Авто** = будущий **агент автообновления** кассового ПО / конфига (фаза 5 плана), который сам раскатывает типовые обновления на объекты.

| Typical | Смысл |
|---------|--------|
| №21 Раскатка ПО | Сейчас руками; когда агент заработает — **успешная** массовая раскатка **вне** премии инженера |
| №23 Автообновление: сбой / офлайн / доводка | То, что **останется человеку**: точка не обновилась, offline, ручной дожим |

Пока агента нет — №21 и №23 оба ручные; №23 заводим заранее, чтобы потом не сваливать хвост в «Доп. настройки».

## Ответ: отличие №6 и №18 (банк)

| # | Typical | Когда выбирать | База (после правок) |
|---|---------|----------------|---------------------|
| 6 | **Оплата не проходит** (банк / процессинг / ДУЭТ) | Инцидент: на кассе **не проходит оплата** прямо сейчас | **5** |
| 18 | **Банк / модуль банка (ручная доводка)** | Работа с **ПО/модулем банка**: установка, настройка, отчёты банка, не «клиент не может оплатить» | **5** (было 15) |

При сомнении: если кассир звонит «не бьёт карта / ДУЭТ» → №6; если «нужно поставить/обновить банковский модуль на N касс» → №18 (+ ×N при пакете).

## Разделение №15 (важно)

Согласующий: типовые **расхождения при закрытии смены** — большой % заявок, должны быть **5**, отдельно от кастомных разборов.

**Предложение в каталог:**

| Typical | База | Когда |
|---------|-----:|-------|
| **Расхождение смены / принуд. закрытие** (оперативное) | **5** | типовой сценарий закрытия смены, знакомый разбор |
| **Отчёты** (вопросы / не печатается отчёт) | **5** | без глубокого расследования |
| **Расхождение / расследование** (№24) | **30** | кастом: 1С, ОФД, ЧЗ-отклонения, «почему не бьётся» сутками |

Имя «Отчёты и расхождения данных» как один catch-all **не оставляем** — иначе снова смешают типовое и разбор.

## Правка имён раскаток и слияния (2026-07-24)

| Было | Стало |
|------|--------|
| Массовое обновление… | **Обновление / раскатка ПО** · **Обновление НДС…** (без «массовое») |
| Автообновление: сбой… | **не заводим** → № обновления ПО |
| Сканер, весы | **Периферия** (сканер, весы, экран покупателя, считыватель карт…) |
| Доступ/код + Настройки и доступы | **одно:** Доступы и настройки (код кассира, портал, не блокирует) |
| Сопутствующий сервис клиента | **убрать** → Нестандарт + Суть |
| Оборудование + Запуск/подготовка | **одно:** Подключение / настройка / запуск объекта (или партии) |

Итого typical: **22** пункта. См. [22](22-typical-base-weights.md).
