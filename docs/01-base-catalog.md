# Справочник: тип работ → базовые баллы

Инженер выбирает **одну базовую категорию** на заявку (или на логический этап, если так решим в Okdesk). Баллы из этого файла — **норма по умолчанию**, не оценка «как тяжело мне было».

Ориентиры из текущей практики (стартовые значения, подлежат ревизии):

| Код | Категория | Базовые баллы | Комментарий |
|-----|-----------|---------------|-------------|
| `T5` | Типовая / понятное решение | 5 | Повторяющиеся, известный сценарий |
| `C15` | Консультация / средняя сложность | 15 | Разбор без глубокой настройки сервисов |
| `S30` | Сложная настройка сервиса (ЧЗ, ПИОТ и аналоги) | 30 | Штатный сценарий настройки **без** затяжных сопутствующих проблем |

## Правила выбора

1. Категория отражает **характер работ**, а не желаемый размер премии.
2. Если уложились в обычный сценарий категории — только база, без надбавок.
3. Если борьба ушла в сопутствующие проблемы (сервис, ОС клиента, полдня вместо часа) — база + [осложнение](02-complications.md), а не «ставлю 60 от себя».
4. Несколько мелких типовых действий в одной заявке → как правило одна категория `T5` (не плодить баллы дроблением), если иное не описано отдельно.

## Что нужно согласовать

- Полный список категорий под ваш реальный микс (не только 5 / 15 / 30).
- Примеры «это точно T5 / C15 / S30» для обучения команды.
- Маппинг на поля Okdesk (обязательный выпадающий список).

См. также [open-questions.md](open-questions.md).
