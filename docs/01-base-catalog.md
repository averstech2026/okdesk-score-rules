# Справочник: тип работ → базовые баллы

Инженер не выбирает число вручную: база берётся из **типовой проблемы** (маппинг в [22-typical-base-weights.md](22-typical-base-weights.md)) или из **типа заявки Нестандарт** → 15.

| Код | Базовые баллы | Смысл |
|-----|-------------:|-------|
| `T5` | 5 | Типовой инцидент / известный сценарий |
| `C15` | 15 | Тип **Нестандарт** (+ редкие случаи) |
| `S30` | 30 | Сложный разбор / статья БЗ |
| `H60` | 60 | Тяжёлый typical **без** выезда (оборудование, запуск…) |

**Утверждено 2026-07-24:** четыре кода; `H60` — база для тяжёлых typical.  
Имена typical в Okdesk **без** префиксов `[5]`/`[30]`/`[60]` (чтобы не тянуть выбор).

Целевой список: **22** пункта — [22](22-typical-base-weights.md) · [typical-final.html](../analysis/typical-final.html).

## Правила выбора

1. Категория отражает **характер работ**, а не желаемый размер премии.
2. Обычный сценарий категории — только база, без надбавок.
3. Сопутствующие проблемы → база + [осложнение](02-complications.md) **+15 или +30** (не «ставлю другое число»).
4. Несколько мелких типовых в одной заявке → как правило одна `T5`.
5. Нет подходящего пункта → тип **Нестандарт** + Суть (не «ближайший» typical).

### Выезд

При чекбоксе `departure` баллы заявки = **только 60**, без typical / N / осложнения.  
Даже если typical с маппингом H60 — при выезде **не** 60+60 и **не** H60+осложнение. См. [03](03-field-visits.md).

### Нестандарт

Тип заявки `nonstandard` → база **15**. Поле `typical` **не показываем** / не обязателен. Обязательны Суть + способ. См. [12](12-nonstandard-issue-type.md).

### Пакет по объектам

```
баллы = база_категории × N_объектов  (+ дежурство; выезд — см. выше)
```

`N` явно в заявке (`object_count` + список объектов). Пакетные typical: банк-модуль, обновление ПО, обновление НДС, запуск партии.

См. также [22](22-typical-base-weights.md) · [21](21-okdesk-new-fields.md) · [25](25-solution-final.md).
