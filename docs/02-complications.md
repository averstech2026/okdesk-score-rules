# Надбавки за осложнение

Нужны, чтобы тяжёлый кейс внутри той же категории (например ПИОТ на полдня из‑за Windows/сервиса) не проигрывал шести типовым заявкам по 5 баллов.

## Принцип

- Список надбавок **закрытый**.
- К каждой надбавке — **краткая причина** (и желательно след в переписке/времени в Okdesk).
- Дорогие комбинации (база + надбавки выше порога) идут в **выборочный аудит**.

## Черновик справочника надбавок

| Код | Условие | Баллы | Когда применять |
|-----|---------|-------|-----------------|
| `X_SIDE` | Сопутствующие проблемы вне основного сервиса (ОС, драйверы, сеть, сторонний софт) | +15 | Без этого основная настройка не завершается |
| `X_LONG` | Объективно длительная работа в рамках одной заявки | +15 | Ориентир: существенно дольше нормы категории (порог часов — согласовать) |
| `X_ESC` | Эскалация / привлечение 2-й линии / вендора | +15 | Зафиксировано в заявке |
| `X_REOPEN` | Повторный заход по той же проблеме не по вине инженера (среда клиента, внешний сбой) | +10 | Не путать с возвратом из‑за плохого решения |

Значения и набор кодов — **черновик**, утверждается отдельно.

## Что нельзя

- Складывать надбавки «на всякий случай» без фактов в заявке.
- Заменять надбавкой выбор более дорогой базовой категории, если по сути работа типовая.
- Ставить `X_LONG` только потому что «день был тяжёлый» без привязки к этой заявке.

## Потолок (предложение)

Сумма надбавок на одну заявку не больше **+30** (или не больше базового веса — выбрать при утверждении), чтобы не вернуть свободную накрутку под видом флажков.

## Связь с аудитом

Обязательный или приоритетный разбор, если:

- база `S30` + любая надбавка;
- сумма по заявке ≥ 45 (без учёта выезда и ×2);
- доля заявок с надбавками у сотрудника заметно выше медианы сегмента.
