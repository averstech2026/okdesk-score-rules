# Надбавка за осложнение (усложнение)

Нужна, чтобы тяжёлый кейс внутри той же категории (например ПИОТ на полдня из‑за Windows/сервиса) не проигрывал шести типовым заявкам по 5 баллов.

**Не применяется**, если в заявке стоит чекбокс выезда — тогда баллы = только **60** (см. [03-field-visits.md](03-field-visits.md)).

## Поля в Okdesk

В диалоге «Новый дополнительный атрибут» → **Тип атрибута** — заводим **два** поля:

| # | Название (пример) | Код (пример) | **Тип атрибута** в Okdesk | Значения |
|---|-------------------|--------------|--------------------------|----------|
| 1 | Осложнение | `complication_level` | **Значение из списка** | `+15` · `+30` |
| 2 | Описание осложнения | `complication` | **Строка** | свободный текст |

**Не путать:**

| Тип в Okdesk | Зачем |
|--------------|--------|
| **Значение из списка** | Один выбор ← поле №1 |
| Набор значений из списка | Мультивыбор — **не** нужен |
| Строка | Текст причины ← поле №2 |
| Чекбокс | Не используем для осложнения |

### Когда начисляется надбавка

| Условие | Баллы осложнения | Отчёт |
|---------|-----------------:|-------|
| Выбран уровень (**+15** или **+30**) **и** описание **не пустое** | +15 или +30 | норма |
| Уровень выбран, описание **пустое** | **0** (надбавки нет) | **красный флаг** «осложнение без описания» |
| Описание есть, уровень не выбран | **0** | жёлтый/красный флаг «описание без уровня» (по желанию) |
| Оба пустые | 0 | — |

Без Expert нельзя сделать «уровень → описание обязательно» в Okdesk — поэтому правило в **формуле премии + отчёт**: нет текста → нет денег; с текстом без уровня / уровень без текста → подсветка.

```
если выезд:
  баллы = 60 × (1|2)
иначе если уровень ∈ {+15,+30} И описание_непустое:
  баллы = (база × N + уровень) × (1|2)
иначе:
  баллы = (база × N) × (1|2)   // уровень без описания → без надбавки
```

Выезд и работа: при выезде typical не суммируется (см. [03](03-field-visits.md)).

## Уровни (когда уместны)

| Значение | Баллы | Когда |
|----------|------:|-------|
| (не выбрано) | 0 | Типовой сценарий |
| **+15** | +15 | Сопутствующая проблема / умеренно дольше нормы / короткая эскалация |
| **+30** | +30 | Полдня / многочасовой разбор / долгая эскалация / вендор |

В описании — одна фраза по делу (не «сложно» / «ок»).

### Что нельзя

- Ставить уровень «на всякий случай».
- Заменять уровнем выбор более дорогой базовой категории, если работа типовая.
- Подменять уровнем корректный **N** на пакетной раскатке.
- Ставить на выездной заявке «вдобавок» к 60 (на премию не влияет).
- Ставить на MFC-техbulk (если такие ещё останутся).

### Аудит

Приоритетный разбор / красный в отчёте, если:

- выбран уровень (+15/+30), а **описание пустое** → надбавка **0**, строка красным;
- `S30` + валидное осложнение (уровень + описание);
- доля **+30** заметно выше медианы сегмента;
- описание-мусор («ок», «1») при начисленной надбавке — снять при аудите.
