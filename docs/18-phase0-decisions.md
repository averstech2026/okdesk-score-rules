| Поле | Тип | Обязательность (регламент) | Решение |
|------|-----|----------------------------|---------|
| `typical` | select | да (как сейчас) | источник **базы** через маппинг → T5/C15/S30 |

| **N объектов** | число **или** список в описании | для пакетных typical | см. §0.4 |

Без Expert: условная обязательность («если чекбокс — то поле») **недоступна** → регламент + аудит.

## 0.4 Как задаём N объектов

| Вариант | Плюс | Минус |
|---------|------|-------|
| **A. Атрибут «Число объектов (N)»** | удобно в отчётах и тени | без Expert не обязателен «если typical пакетный» |
| B. Только список в описании | без нового поля | хуже автоматика; N иногда «на глаз» |
| **C. Атрибут + список в описании (рекомендация)** | отчёт + аудит | чуть больше заполнения |

**Предложение:** вариант **C**. Без N на пакетном typical → флаг аудита, не свободные 300–1000.

Решение: [ ] C / [ ] A / [ ] B
