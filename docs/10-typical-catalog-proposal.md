## Диагноз

| Контур | Заявок | Другое + пусто |
|--------|-------:|---------------:|
| Вся выборка | 19 640 | 37% |
| Только MFC | 6 574 | **67%** |
| Без MFC (нативнее) | 13 066 | **22%** |

- Комментарий «Техническое закрытие» — про процесс переноса/архивации, **не** про качество справочника.
- Каталог и KPI заполнения `typical` считать **без MFC-bulk**.
- Без MFC лидер — «Доп. настройки…» (catch-all); «Другое» всё ещё ~1 250 — там искать пробелы (НДС, раскатки, карты/клиенты, сервисы вроде Work Bright).
- «Не обновились лимиты» (19) — кандидат на слияние в «Карты и клиенты».
- Крупные баллы часто = **пакет по объектам** (раскатка/НДС/запуск), а не «на глаз».

## Предлагаемые объединения

| Было | Станет |
|------|--------|
| Оплата ДУЭТ + банк/процессинг | Оплата не проходит (банк / процессинг / ДУЭТ) |

| Консультация | Консультация / звонок |
| Не работает ЧЗ | ЧЗ / маркировка (оперативный инцидент) |

## Пайплайн реструктуризации typical (шаг 2 плана)

Порядок внедрения в Okdesk:

```
1. Слить дубли (таблица выше)
2. Вынести catch-all «Доп. настройки» / «Другое» в явные пункты
3. Добавить блок «раскатки и ручной хвост» (ниже) с правилом база×N
4. Параллельно тип «Нестандарт» + Суть (docs/12)
5. Убрать «Другое» из списка после 1–2 недель параллели
```

### Блок A — массовые раскатки (сейчас ручные, цель → автообновление)

| Typical (новый/явный) | База | Правило N | Комментарий |
|-----------------------|------|-----------|-------------|
| **Обновление / раскатка ПО** (не путать с установкой) | T5 | **`× N`** (N≥1) | Спулер, SDK, лого, frontupdater, апдейт Фронт/ЧЗ. Без слова «массовое» — одна точка тоже этот пункт. После автоагента успешная — вне премии; сбой/доводка — снова №21 |
| **Обновление НДС / фискальных параметров** | T5 | **`× N`** (N≥1) | Кампания или одна точка; не путать с «НДС: точечная ошибка ставки» |

В заявке: список/таблица объектов или поле «Число объектов (N)». Без N — на аудит, не свободные 300–1000.

### Блок B — ручной хвост (без отдельного «фейл авто»)

| Typical (новый) | База | N? | Когда выбирать |
|-----------------|------|----|----------------|
| **НДС: точечная ошибка ставки / режима** | T5 | обычно 1; если несколько касс — `× N` | «На чеках 20%, надо 22%» — не кампания обновления параметров |
| **Банк / эквайринг / модуль банка (ручная доводка)** | T5 | `× N` при пакете | Установка/настройка банковского ПО |
| **Запуск / подготовка объекта (или партии)** | H60 | **`× N`** | Подготовка/запуск; не «раскатка уже работающего ПО» |

Отдельный typical «Автообновление: сбой…» **не заводим**: пока/когда агент не закрыл точку — инженер берёт **Обновление / раскатка ПО** (№21).

### Блок C — прочие добавления (как планировали)

| Typical | Заметка |
|---------|---------|
| **Сопутствующий сервис клиента** (Work Bright / AllBusiness…) | Не путать с «Доп. настройки» |
| **Настройки и доступы (не блокирует)** | Сжатый остаток бывшего «Доп. настройки» |
| **Оборудование / запуск** | Связать с блоком B «Запуск / подготовка» (не плодить два почти одинаковых — выбрать одно имя при утверждении) |

### Связь с нормами баллов

```
пакетный typical ∈ {раскатка ПО, обновление НДС, запуск партии}:
  баллы = база × N_объектов  (N≥1)  (+ выезд / ×2 дежурство по правилам)

точечный typical (инцидент НДС, консультация, одна касса):
  баллы = база  (N=1)
```

См. [01-base-catalog.md](01-base-catalog.md). Симуляция премий: `scripts/compare_bonus_models.py`, `scripts/analyze_mass_updates.py`.

### Что проверить при утверждении с тимлидами

- [ ] Одно имя для «Запуск / подготовка» vs «Оборудование / запуск»
- [ ] База раскатки: T5 или C15 (и та же для НДС-кампании?)
- [ ] Поле N в Okdesk: отдельный атрибут vs список в описании (регламент)
- [ ] После автоагента: успешная раскатка — без премии инженеру / сервисная заявка; в премии только блок B
- [ ] «Оплата (банк…)» остаётся инцидентом оплаты; блок B «Банк / модуль банка» — про ПО/настройку модуля, не «не проходит оплата»

## Предлагаемые добавления (краткий чеклист в Okdesk)

1. Обновление / раскатка ПО (`× N`, N≥1)
2. Обновление НДС / фискальных параметров (`× N`, N≥1)
3. НДС: точечная ошибка ставки / режима
4. Банк / модуль банка — ручная доводка (`× N` при пакете)
5. Запуск / подготовка объекта или партии (`× N`)
6. ~~Автообновление: сбой…~~ — **не заводим** (сбой → п.1)
7. Сопутствующий сервис клиента (Work Bright / AllBusiness…)
8. Сжатые «Настройки и доступы»
9. **Статья базы знаний** — см. [22-typical-base-weights.md](22-typical-base-weights.md)

## Что внутри двух корзин (кратко)

### «Другое» (3639; без MFC 1248)
Карты/клиенты, звонки, чеки/удаления, расхождения, обмены, Work Bright/AllBusiness, НДС, письма по ЧЗ, хвост нераспознанного → разнести по блокам A/B/C и Нестандарт.

### «Доп. настройки, сопутствующий сервис…» (2203)
Внутри: карты/дотации (~464), обмены/меню (~221), доступы (~122), раскатки ПО (~71), код кассира/портал, ЧЗ/ПИОТ, запуски. Пункт **сузить** до доступов/точечных настроек; раскатки и запуски — в блок A/B.

Подробные таблицы — в `analysis/typical-catalog.html`.
