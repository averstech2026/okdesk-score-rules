# Выезды на объекты

Выезд оценивается **отдельно** от удалённой работы: фикс зависит от зоны удалённости (логистика), а работы на месте — по [базовому каталогу](01-base-catalog.md) и при необходимости [осложнениям](02-complications.md).

## Баллы за зону (действующая практика)

| Зона | Баллы |
|------|------:|
| Москва | 30 |
| Ближнее Подмосковье | 60 |
| Дальнее Подмосковье | 120 |

Границы «ближнее / дальнее» зафиксировать однозначно (список городов / км / внутренний справочник).

## Когда выезд допустим

Закрытый список причин (инженер выбирает одну):

| Код | Причина |
|-----|---------|
| `V_NO_NET` | Нет интернета / нельзя сделать удалённо |
| `V_HW` | Замена / установка оборудования на месте |
| `V_CLIENT` | Требование клиента (в т.ч. «уважаемый клиент») |
| `V_OTHER` | Иное (краткий комментарий обязателен) |

## Начисление

```
баллы_заявки_с_выездом =
  баллы_зоны
  + базовый_вес_работ_на_месте
  + надбавки_осложнения
  × коэффициент_дежурства
```

Если на выезде только доставка/замена без отдельной «сложной» настройки — база может быть `T5`/`C15` по факту работ, плюс зона.

## Контроль

Флаги для разбора:

- много `V_CLIENT` / `V_OTHER` у одного сотрудника;
- выезд в день, когда по тому же объекту уже была успешная удалённая сессия без смены обстоятельств;
- доля баллов от выездов сильно выше коллег в том же сегменте.

## Дежурство

×2 на выездные баллы — **только** если выезд выполнен в дежурную смену по табелю (см. [04-duty.md](04-duty.md)).
