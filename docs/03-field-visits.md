# Выезды на объекты

## Правило начисления (утверждено)

Если стоит чекбокс выезда (`departure`) — баллы заявки = **только 60**.

```
если departure:
  баллы = 60 × (2 если дежурство по табелю, иначе 1)
  // typical, N, осложнение — НЕ суммируются
иначе:
  баллы = (база_из_typical × N + осложнение 0|15|30) × (1|2)
```

Выезд — это **не** осложнение и **не** «база работ + 60». Фикс **60** заменяет нормативную базу по typical для этой заявки.

| Условие | Баллы до ×2 |
|---------|------------:|
| `departure` = нет | по правилам без выезда (база × N + осложнение) |
| `departure` = да | **60** (игнорировать typical / N / осложнение) |

Зона (Москва / ближнее / дальнее) **не влияет на баллы**.

Typical и способ решения на выездной заявке по-прежнему заполняют для учёта/аналитики — но **в премию по баллам не входят**.

## Когда выезд допустим

Отдельное поле «причина выезда» **не заводим**. Достаточно чекбокса `departure` и контекста в описании/переписке при необходимости аудита.

## Контроль

Флаги для разбора:

- много выездов у одного сотрудника без явного контекста в заявке;
- выезд в день, когда по тому же объекту уже была успешная удалённая сессия без смены обстоятельств;
- доля баллов от выездов сильно выше коллег в том же сегменте.

## Дежурство

×2 на **60** — только если выезд в дежурную смену по табелю (см. [04-duty.md](04-duty.md)) → **120**.
