# Принципы

## Что сохраняем

- Оценка труда в **баллах** (привычная шкала для команды и премии).
- Повышающий коэффициент за **дежурную смену** (×2).
- Фикс за **выезд**: если чекбокс выезда — **только 60** (не сумма с typical).

## Что меняем

1. Инженер **не вписывает баллы вручную**. Он выбирает **категорию** из справочника (и при необходимости уровень осложнения + описание). Баллы считает правило. При выезде — достаточно чекбокса выезда.
2. Один тип работ имеет **базовый** вес. Если работа объективно тяжелее нормы — **осложнение +15 или +30** (поле-**список** в Okdesk) и краткое описание, а не свободная цифра баллов.
3. **Конвейер** и **общий пул** — разные режимы труда. Их нельзя честно сравнивать одной абсолютной суммой баллов без поправки или сегментации.
4. **SLA группы** ≠ персональный KPI. Персональная ответственность — с момента назначения/взятия заявки.
5. Внеурочная работа вне дежурства **пока не выделяется** отдельным коэффициентом (нет явной боли). При необходимости — позже.

## Слои начисления

```
если выезд (departure):
  баллы = 60 × (1 или 2)
  // typical, N, осложнение не входят в сумму

иначе:
  баллы = (база категории × N + осложнение 0|15|30) × (1 или 2)
```

## Анти-стимулы, которых избегаем

| Плохой стимул | Как закрываем |
|---------------|---------------|
| «Проще сделать 6 типовых, чем 1 тяжёлый ПИОТ» | Надбавки за осложнение + аудит дорогих кейсов |
| «Выгоднее сесть на конвейер 50 шт/день» | Сегменты / нормы / вклад относительно своего потока |
| «Нарисую 30 баллов» | Нет свободного ввода баллов |
| «Штраф за просрочку, которую никто не брал» | SLA группы → команда/владелец пула; лично — после взятия |

## Роли

| Роль | Зона |
|------|------|
| Инженер | Категория (если не выезд), осложнение (+15/+30) и описание, чекбокс выезда |
| Владелец пула на смену / тимлид | Очередь группы не гниёт; разбор красного SLA |
| Руководитель | Утверждение справочника, выборочный аудит, премия, баланс потоков |
