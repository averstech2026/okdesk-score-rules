## Что спрятано в «Другое» (способ), без MFC

Частые заголовки при *только* «Другое»: звонок, вопрос по кассе, проблема с кассой, не печатаются/не выходят чеки, начисление средств, код кассира, расхождение, fix меню, клиенты, доступ, зависла касса, ЧЗ-письма…

То есть «Другое» в решении = «не нашли глагол в списке» или «и так сойдёт», а не отдельный класс работ.

## Предложения по оптимизации

### Сжать / объединить

| Было | Станет | Зачем |
|------|--------|-------|
| Рестарт ПО + Рестарт GriServ | **Рестарт ПО / сервиса** (в т.ч. GriServ, спулер, фронт, IIS) | GriServ — частный случай; можно подписью в скобках |
| Провода + питание вкл/выкл | **Железо: питание / кабели / переподключение** | Один класс «потрогать железо» |
| Выполнить обмен + SALES3… | **Обмен / перевыгрузка данных** (точки, SALES3, смены) | SALES3 редко, смысл тот же |
| Бумага → в печать/ФР или в железо | Убрать отдельным | 16 раз |

### Расширить / добавить (закрыть дыры «Другое»)

1. **Консультация / ответ без изменения системы** — сейчас часто «Другое» или «Самовосстановление».
2. **Массовое обновление / раскатка ПО** — спулер, ЛМ2, модули (сейчас «Другое» или «Настройка ПО»). Согласовать с typical блока A в `docs/10` (`база × N`).
3. **Работа с картами / лимитами / начислениями** — начисление, дотация, клиенты.
4. **Изменение фискальных параметров / НДС (кампания)** — массовый пуш ставок; не путать с точечной правкой.
5. **Точечная правка ставки / режима НДС** — одна (или мало) касс; typical «НДС: точечная…».
6. **Доводка после автообновления (фейл / офлайн)** — ручной хвост агента; typical из блока B в `docs/10`.
7. **Диагностика без устранения / эскалация** (опционально) — если часто «спросили и ушли».

Стыковка с typical: пакетные способы ↔ пакетные проблемы; после автоагента успешная раскатка не должна идти в премию инженера как «раскатка», только фейл/доводка.

«Самовосстановление» оставить, но не путать с консультацией: самовосстановление = проблема ушла сама; консультация = объяснили без фикса.

### Сузить catch-all

**«Настройка ПО и предоставление доступов»** — слишком широкий (725), рядом с typical «Доп. настройки».  
Разделить или явно сузить:

- **Выдача доступа / прав / код кассира**
- **Точечная настройка конфига** (не раскатка)

### «Другое» в способах

Аналогично typical:

- убрать из основного списка или  
- тип заявки «Нестандарт» + обязательная «Суть» покрывает и проблему, и решение одним текстом, **или**  
- отдельное уточнение способа при «Другое».

Пока без Expert — KPI: доля «только Другое» в способах **без MFC**.

## Целевой черновик (~12–14 пунктов)

## Связь с каталогом проблем

Многие пары typical↔solution уже хорошо бьются (смена→принуд. закрытие; отменить чек→отменить чек; меню→обмен).  
Оптимизацию **типовых проблем** и **способов** лучше катить вместе: иначе снова будет «проблема точная, решение Другое».
