Что было → что станет

Убираем самооценку баллов «на глаз» (select 5…1000). Вводим нормы: база из типовой проблемы, пакет × N, осложнение +15 или +30 (поле-список Okdesk), выезд = только 60 (без суммы с typical). Курс: 1 балл = 15 ₽.

Статус: черновик на утверждение Граница фазы 0 — согласовать таблицы ниже. Фаза 1 = поля в Okdesk при старых выплатах. Платим по формуле только с даты X (после тени 4–6 недель).

1. Модель баллов и премии

Было → станет ключевое
ТемаБылоСтанет
Источник баллов Инженер выбирает число в ticket_weight (select 5…1000) Баллы считает правило: база из typical + пакет × N + осложнение + выезд, затем ×2 дежурства
Ориентиры 5/15/30 Неформальные Коды T5=5 / C15=15 / S30=30 — единственные базы
Тяжёлый кейс Часто «ставлю 60 сам» или недооценка База категории + осложнение +15 или +30
Пакет (N касс) Одна большая цифра «на глаз» (300–1000) база × N, N видно в заявке
Выезд Зоны 30 / 60 / 120 Только 60 при чекбоксе (typical не суммировать)
Курс 1 балл = 15 ₽ Без изменений
Статьи премии Сумма баллов; дежурства отдельным циклом ~1,5 нед Две статьи: обычные / дежурство; дежурство сверять с табелем
Когда платим — Только с даты X (после тени 4–6 нед)

Формула v1

если выезд (departure):
  баллы = 60 × (2 если дежурство, иначе 1)
  // typical / N / осложнение НЕ суммируются

иначе:
  баллы = (база(T5|C15|S30) × N
           + осложнение только если уровень И непустое описание
          ) × (2 если дежурство, иначе 1)
  // уровень без описания → надбавка 0 + красный в отчёте

премия_₽ = сумма_баллов × 15

2. Поля Okdesk

ПолеБылоСтанет
ticket_weight Свободный select 5…1000 — главный источник премии До даты X — как сейчас; после X — заморозка / служебный расчёт
typical Обязательный select; много «Другое» / «Доп. настройки» Сжатый каталог + явные раскатки/хвост; источник базы
solution_method Multiselect; «Другое» доминирует Сжатый список вместе с typical; для аудита, не источник баллов v1
Осложнение Нет Два атрибута: «Значение из списка» (+15/+30) + «Строка»; надбавка только если оба; иначе 0 + красный в отчёте
departure Checkbox + зона Checkbox → баллы заявки = только 60
watch Checkbox ×2 вручную Остаётся; истина — табель; статья «дежурство» отдельно
N объектов Нет поля; объём прячут в баллах Атрибут «Число объектов» + список в описании
Нестандарт Нет (дыра = «Другое») Тип + поле «Суть»; не для MFC-bulk
Expert Нет автоправил По-прежнему нет → регламент + аудит

3. Осложнение (новый слой)

ТемаБылоСтанет
Учёт тяжести Свободная цифра в баллах Поле-список Okdesk: +15 или +30 (+ строка описания)
Почему тяжело Часто не видно Текст описания осложнения
Суммирование — Нет: одно значение в списке; на выезде осложнение не влияет
Не путать Осложнение ≠ объём пакета (N) ≠ выезд ≠ дежурство.

4. Типовые проблемы (typical)

4.1 Слияния дублей

БылоСтанет
Оплата ДУЭТ + отдельно банк/процессинг Оплата не проходит (банк / процессинг / ДУЭТ)
Вопросы по отчётам + не печатается отчёт (+ сверки) Отчёты и расхождения данных
Карта клиента + лимиты + «клиенты» из Другое Карты питания и клиенты
Отменить чек Корректировка чека (сторно, возврат, удаление, перенос)
Консультация Консультация / звонок
Не работает ЧЗ ЧЗ / маркировка (оперативный инцидент) ≠ расследование отклонений

4.2 Catch-all и дыры

БылоСтанет
«Другое» (~37% всей выборки; дыра жива и без MFC) Убрать после 1–2 нед параллели; остаток → Нестандарт + Суть
«Доп. настройки, сопутствующий сервис…» — широкая свалка Сузить до настроек и доступов; раскатки/запуски/сервисы — отдельно
НДС, раскатки, запуски, банк-модуль спрятаны в catch-all Явные пункты (таблица ниже)

4.3 Новые / явные пункты (раскатки и хвост)

Typical (станет) База Объём Комментарий
Массовое обновление / раскатка ПО T5 × N Сейчас часто «на глаз» 100–1000
Массовое обновление НДС / фиск. параметров T5 × N Кампания конфига, не точечный инцидент
НДС: точечная ошибка ставки / режима T5 / C15 обычно 1 Ручной хвост после авто
Банк / модуль банка (ручная доводка) C15 × N при пакете Не путать с «оплата не проходит»
Запуск / подготовка объекта (или партии) S30 или C15 × N Одно имя; не плодить «Оборудование / запуск»
Автообновление: сбой / офлайн / доводка T5 / C15 × N После агента — это в премии
Сопутствующий сервис клиента C15 1 Work Bright / AllBusiness…
Настройки и доступы (не блокирует) T5 1 Остаток бывших «Доп. настроек»

4.4 Автообновление

БылоСтанет
Ручные массовые раскатки ПО в премии инженера (~15% фонда баллов) Агент делает успешную раскатку; вне премии инженера
Фейлы/офлайн смешаны с общей раскаткой Отдельный typical «сбой / офлайн / доводка» × N

5. Способы решения (solution_method)

БылоСтанет
«Другое» — доминирующий выбор Убрать после параллели; дыры — явными пунктами
Рестарт ПО + отдельно GriServ Рестарт ПО / сервиса (в т.ч. GriServ)
Провода + питание Железо: питание / кабели / переподключение
Обмен + SALES3 Обмен / перевыгрузка данных
«Проверить бумагу» (мёртвый) Убрать
«Настройка ПО и доступы» — слишком широко Доступ / код кассира + точечная настройка конфига
Нет: консультация, раскатка, карты, НДС, доводка авто Добавить (см. docs/13)

8. Выезды и дежурство

ТемаБылоСтанет
Баллы выезда Мск 30 / ближнее 60 / дальнее 120; «крупная» сумма Только 60 при departure
Typical при выезде Могли суммировать с выездом Не входят в формулу баллов (только учёт)
Дежурство ×2 Чекбокс watch, риск рассинхрона ×2 к 60 → 120 по табелю; две статьи в отчёте

9. Что не меняем в фазе 0–1

  • Курс 15 ₽ / балл
  • Идея баллов и премии за закрытые заявки
  • Коэффициент дежурства ×2
  • Текущие выплаты до даты X (фаза 1–2 = каталоги + тень)
