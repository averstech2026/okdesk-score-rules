Что скрывалось в «Доп. настройки, сопутствующий сервис…»

2 203 заявки — второй catch-all. Почти все без MFC (2 061). Название обещает «неблокирующий сопутствующий сервис», но по факту туда кладут разное: карты/дотации, обмены, раскатки ПО, коды кассира, ЧЗ-модули, запуски. Медиана баллов 10, среднее ~23.

Важное различие «Сопутствующий сервис» в текущем пункте Okdesk ≠ только Work Bright. Work Bright / AllBusiness чаще оказывались в «Другое». В «Доп. настройках» — в основном внутренние работы по кассе/интеграциям без точного пункта.

Пункт Откуда Что входит / что вытаскиваем из корзин Ориентир
Как внедрять (проблемы)
  1. Вынести из корзин: карты/клиенты, раскатка ПО, НДС, сопутствующий сервис клиента; слить ДУЭТ с оплатой.
  2. «Доп. настройки…» сузить → «Настройки и доступы (не блокирует продажу)».
  3. «Другое» в проблемах → тип заявки «Нестандарт» + обязательная «Суть» (без Expert). KPI доли — без MFC.
  4. Способ решения при нестандарте всё равно выбирается из каталога способов (см. ниже).

Тип «Нестандарт» — только про проблему

Имеет смысл, когда симптом/ситуация не укладывается в каталог типовых проблем. Это не значит «мы не знаем, что сделали»: действие инженера почти всегда есть в списке способов.

Проблема

Нет честного пункта в typical → тип Нестандарт + поле «Суть нестандарта» (обязательно при «Решена»).

Убрать «Другое» из выпадающего списка проблем. MFC-перенос этим типом не кормить.

Способ решения

Всегда из каталога способов: рестарт, обмен, доступ, консультация, раскатка, железо…

Нестандартная заявка ≠ неизвестный способ. Целевой список способов как раз закрывает дыры нынешнего «Другое» в solution.

Связка с проблемами Если typical = «Доп. настройки» / «Другое» / «Консультация», способ почти всегда тоже «Другое». Каталоги менять вместе. Тип «Нестандарт» закрывает дыру в проблеме; дыру в способе закрывает целевой список ниже.
Покроет ли целевой список то, что сейчас в «Другое» (способ)? Основную массу — да. Звонки → консультация; касса зависла → рестарт/железо; чеки → рестарт/БД/сеть; начисления/клиенты → карты; код кассира → доступ; меню/обмен → обмен; расхождение → смена/сверка; ЧЗ-письма → сеть или раскатка; апдейты → раскатка; НДС → фискальные параметры. Останется небольшой хвост разовых тем — для них ближайший способ (консультация / точечная настройка / БД), а нестандартность — в типе проблемы, не в способе.
